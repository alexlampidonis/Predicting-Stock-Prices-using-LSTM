# Predicting Stock Prices using RNNs (LSTM + Optuna + Captum)

This project forecasts the **next-day percentage price movement** of 442 companies using a Long Short-Term Memory (LSTM) recurrent neural network. The model is trained on ~12 years of daily returns (05/04/2010 – 31/03/2022), tuned automatically with **Optuna**, and interpreted afterwards with **Captum's Integrated Gradients** to understand which past days most influence the model's predictions.

The task is framed as a multivariate time-series regression problem: at each time step the model sees the daily % change of *all 442 stocks at once* and learns to predict the % change of all 442 stocks on the *next* day.

## Problem Overview

Given the historical daily percentage returns of 442 companies, the goal is to predict each company's percentage return on a single target day (1 April 2022). This is a classic **multi-output time-series forecasting** problem: the input at each time step is a 442-dimensional vector (one value per company), and the output is also a 442-dimensional vector — the predicted next-day return for every company simultaneously.

## Data

- **`train.csv`**: 442 rows (one per company/`ID`) × 3021 columns, where each column is a trading day between 05/04/2010 and 31/03/2022, and each cell is that company's daily percentage price change.
- **`sample_submission.csv`**: the submission format — one predicted value per company for the target date (1 April 2022).

Because each row is a company and each column is a date, the data is **transposed** early on so that rows represent time steps (days) and columns represent features (companies) — the format an LSTM expects (`[sequence_length, num_features]`).

## Approach

### 1. Data Preparation

- The raw data is loaded and transposed to shape `(3021 days, 442 companies)`.
- The data is split **chronologically** into an 80% training portion and a 20% validation portion (no shuffling, since this is time-series data and shuffling would leak future information into the past).
- A `MinMaxScaler` (scaling to the range `[-1, 1]`) is **fit only on the training split** and then applied to both the training and validation splits. Fitting the scaler on the full dataset would leak information about the validation period's scale into training, so it is deliberately avoided.

### 2. Random Subsequence Sampling

Rather than creating a single fixed set of overlapping windows, training sequences are generated by repeatedly sampling **random contiguous subsequences** of length `seq_length` from the scaled data. Each subsequence provides `X` (the input window) and `y` (the same window, used to extract the target as the next day's values). This acts as a lightweight form of data augmentation and reshuffling of the training signal on every trial/epoch, since a new random set of windows is drawn each time.

### 3. LSTM Model

The model is a standard multi-layer LSTM followed by a linear output layer:

```
Input [batch, seq_length, 442]
   → LSTM(hidden_dim, num_layers, dropout)
   → take output at the last time step
   → Linear(hidden_dim → 442)
   → Output [batch, 442]  (predicted next-day % change for all companies)
```

Hidden and cell states are zero-initialized at the start of each forward pass. The LSTM processes the whole 442-company vector jointly at each time step, which lets it implicitly learn **cross-company relationships** (sector correlations, co-movements, etc.) rather than modeling each stock in isolation.

### 4. Hyperparameter Optimization with Optuna

[Optuna](https://optuna.org/) is used to search over the model and training configuration. The following hyperparameters were tuned:

| Hyperparameter | Search Space |
|---|---|
| `hidden_dim` | 128 – 512 |
| `num_layers` | 1 – 6 |
| `dropout` | 0.2 – 0.6 |
| `learning_rate` | 1e-6 – 1e-1 (log scale) |
| `optimizer` | Adam / RMSprop / SGD |
| `seq_length` | 2 – 30 days |
| `batch_size` | 16 – 128 |
| `num_sequences` | 100 – 600 (random windows sampled per epoch) |

For each trial, Optuna:
1. Builds a fresh LSTM with the sampled hyperparameters.
2. Generates random training and validation subsequences.
3. Trains for up to 200 epochs, using **MSE loss** on the next-day predictions.
4. Reports the validation loss after every epoch, allowing Optuna's pruner to **stop unpromising trials early** (`trial.should_prune()`), which saves compute.
5. Returns the final validation MSE as the objective to minimize.

The study runs a maximum of 60 trials or 10 minutes, whichever comes first, and keeps the hyperparameter set with the lowest validation MSE.

### 5. Final Training & Prediction

Once the best hyperparameters are found:
1. The scaler is re-applied to the **entire dataset** (train + validation combined), since the final model should use all available history.
2. A new LSTM is built with the winning hyperparameters and trained for 50 epochs on random subsequences drawn from the full dataset.
3. The last `seq_length` days of the dataset are fed into the trained model to predict the next day's (1 April 2022) percentage change for all 442 companies.
4. Predictions are inverse-transformed back to the original scale and written to `submission.csv`.

### 6. Model Interpretation with Captum

To understand *what the model is actually paying attention to*, [Captum](https://captum.ai/)'s **Integrated Gradients** method is applied to the trained model.

Integrated Gradients attributes a prediction to each input feature by accumulating gradients along a straight-line path from a baseline input (typically all zeros) to the actual input. In this project:

- The last 20 days of the validation split are used as a sample input sequence.
- For each of the 442 companies, Integrated Gradients computes how much each of the past days contributed to that company's predicted return.
- These per-company attributions are averaged across all 442 companies to get a single **importance score per lag (day)**.
- The result is visualized as a bar chart of "Mean Attribution Strength" vs. "Days Before Prediction."

**Interpretation of the result:** the attribution plot shows a geometric decay — the model relies most heavily on the days **roughly 3–4 days before** the prediction target, with influence tapering off further into the past. This lines up with the intuitive expectation for financial time series: the most recent information tends to be the most predictive, and the influence of older data decays the further back you go.

## Results

- Best hyperparameters and validation MSE are found automatically via the Optuna study (printed at the end of the optimization run — see notebook output for the specific values from your run, since Optuna's search is stochastic and time-boxed).
- Final predictions for 1 April 2022 are written to `submission.csv` in the same format as `sample_submission.csv`.
- Interpretability analysis confirms the model is using a sensible, decaying lag structure rather than an arbitrary or noisy weighting of past days.

   - Optuna objective function

   - Captum Integrated Gradients interpretation

## Key Takeaways

- **Joint modeling across companies**: predicting all 442 stocks with a single LSTM (rather than 442 separate models) lets the network learn cross-stock dependencies for free.
- **Careful train/val splitting**: the chronological split and "fit scaler on train only" approach avoid look-ahead bias, which is critical for time-series problems.
- **Automated tuning**: Optuna's pruning mechanism makes it feasible to search a fairly large hyperparameter space (including architecture depth, sequence length, and optimizer choice) within a limited time budget.
- **Interpretability matters**: Integrated Gradients turns the LSTM from a black box into something that gives an economically sensible signal — recent days matter most, with a smooth decay further back — which builds confidence that the model has learned a reasonable pattern rather than overfitting to noise.

## Possible Improvements

- Add walk-forward / rolling-window cross-validation instead of a single train/validation split.
- Compare against simpler baselines (e.g., naive "predict zero change," ARIMA, or a linear autoregressive model) to quantify how much the LSTM is actually adding.
- Experiment with attention mechanisms or Transformer-based architectures, which can also provide interpretability via attention weights.
- Incorporate exogenous features (volume, sector, macroeconomic indicators) beyond raw price returns.
